Command Reference

compile

▾

Description

Compiles a .zarathustra source file into a flattened R1CS constraint system. The output binary (out) is used by all subsequent commands.

Usage

$ zarathustra compile -i program.zok [--curve bn128]

Options

  • -i, --input — Path to the .zarathustra source file
  • --curve — Target elliptic curve: bn128, bls12_381, bls12_377, bw6_761, pallas, vesta (default: bn128)

Example

# Compile for the BN128 curve (Ethereum)
$ zarathustra compile -i program.zok --curve bn128
Compiled code written to 'out'
Number of constraints: 2

inspect

▾

Description

Inspects a compiled binary and prints detailed information about the constraint system — number of wires, constraints, public inputs, and private inputs.

Usage

$ zarathustra inspect -i out

Options

  • -i, --input — Path to the compiled binary

Example

$ zarathustra inspect -i out
Constraints:  2
Wires:        5
Public In:    1
Private In:   1

check

▾

Description

Checks a .zarathustra source file for syntax and type errors without performing a full compilation. Useful for quick iteration.

Usage

$ zarathustra check -i program.zok

Options

  • -i, --input — Path to the source file

Example

$ zarathustra check -i program.zok
OK

compute-witness

▾

Description

Executes the compiled constraint system with the provided inputs to compute a valid witness. The witness assigns values to every wire in the circuit and is required for proof generation.

Usage

$ zarathustra compute-witness -i out --abi -s abi.json --stdin <<< '["3"]'

Options

  • -i, --input — Path to the compiled binary
  • --abi — Use ABI encoding for inputs/outputs
  • -s, --stdlib — Path to the standard library ABI file
  • --stdin — Read ABI-encoded arguments from stdin
  • --output — Path to write the witness (default: witness)

Example

# With ABI encoding and JSON arguments from stdin
$ zarathustra compute-witness -i out --abi -s abi.json --stdin <<< '["42"]'
Witness written to 'witness'

setup

▾

Description

Generates a proving key and a verification key from a compiled constraint system. The proving key is used to generate proofs; the verification key is used to verify them and to export Solidity verifiers.

Usage

$ zarathustra setup -i out

Options

  • -i, --input — Path to the compiled binary

Example

$ zarathustra setup -i out
Setup complete
Proving key:      proving.key
Verification key: verification.key

universal-setup

▾

Description

Performs the universal structured reference string (SRS) setup for a given curve and size. This is the first phase of the Marlin proving system — a one-time setup per curve that can be reused across all programs.

Usage

$ zarathustra universal-setup --curve bn128 --size 16

Options

  • -c, --curve — Elliptic curve
  • -s, --size — Maximum power of two constraints (e.g. 16 = 2¹⁶ constraints)

Example

$ zarathustra universal-setup --curve bn128 --size 16
Universal setup written to 'universal_setup.dat'

generate-proof

▾

Description

Generates a zero-knowledge proof using a compiled program, a witness, and a proving key. The proof can be verified locally or on-chain via the exported Solidity verifier.

Usage

$ zarathustra generate-proof -i out -w witness -p proving.key

Options

  • -i, --input — Path to compiled binary
  • -w, --witness — Path to the witness file
  • -p, --proving-key-path — Path to the proving key

Example

$ zarathustra generate-proof -i out -w witness -p proving.key
✓ Proof generated
Proof written to 'proof.json'

verify

▾

Description

Verifies a zero-knowledge proof against a verification key. Returns whether the proof is valid or not.

Usage

$ zarathustra verify -v verification.key -j proof.json

Options

  • -v, --verification-key — Path to verification key
  • -j, --proof — Path to proof JSON file

Example

$ zarathustra verify -v verification.key -j proof.json
✓ PROOF VERIFIED SUCCESSFULLY

export-verifier

▾

Description

Generates a Solidity smart contract that can verify Zarathustra proofs on Ethereum. The contract is self-contained and ready to deploy.

Usage

$ zarathustra export-verifier -i verification.key

Options

  • -i, --input — Path to verification key

Example

$ zarathustra export-verifier -i verification.key
Verifier contract written to 'verifier.sol'
Estimated gas per verification: ~250,000

▾

Description

Pretty-prints a proof JSON file in a human-readable format, showing the proof components and public inputs.

Options

Usage

$ zarathustra generate-smtlib2 -i out

Options

  • -i, --input — Path to compiled binary

Example

$ zarathustra generate-smtlib2 -i out

profile

▾

Description

Profiles a compiled program and shows a breakdown of constraints by operation type. Helps optimize circuits for size and proving time.

Usage

$ zarathustra profile -i out

Options

  • -i, --input — Path to compiled binary

Example

$ zarathustra profile -i out

  Multiplication:  12 constraints
  Assertion:        2 constraints
  SHA256:        1568 constraints
  Total:         1582 constraints

mpc — Multi-Party Computation

▾

Description

Runs a multi-party computation ceremony for distributed trusted setup. Multiple participants contribute entropy to the setup parameters, ensuring that no single party can compromise the setup.

Subcommands

  • mpc init — Initialize the MPC ceremony from a compiled program
  • mpc contribute — Contribute entropy to the ceremony
  • mpc verify — Verify a contribution
  • mpc beacon — Apply a random beacon (e.g. from a blockchain)
  • mpc export — Export the final parameters

Usage

$ zarathustra mpc init -i out
$ zarathustra mpc contribute -i state
$ zarathustra mpc verify -i state
$ zarathustra mpc export -i state -o params

nova — Nova Recursive SNARKs

▾

Description

Nova is a recursive zero-knowledge proving scheme (IVC — Incrementally Verifiable Computation). It allows folding multiple proof steps into a single compact proof. The Nova module supports setup, proving, verifying, and compressing folding proofs.

Subcommands

  • nova setup — Generate Nova proving and verification parameters
  • nova prove — Generate a Nova proof
  • nova verify — Verify a Nova proof
  • nova compress — Compress a folding proof into a more compact SNARK

Usage

$ zarathustra nova setup -i out
$ zarathustra nova prove -i out -w witness
$ zarathustra nova verify -v vk -j proof.json
$ zarathustra nova compress -i proof.json

Quick Start

Full Pipeline — One Shot

$ zarathustra compile -i program.zok --curve bn128
$ zarathustra setup -i out
$ zarathustra compute-witness -i out --abi -s abi.json --stdin <<< '["42"]'
$ zarathustra generate-proof -i out -w witness -p proving.key
$ zarathustra verify -v verification.key -j proof.json
$ zarathustra export-verifier -i verification.key
✓ verifier.sol — ready to deploy on Ethereum
